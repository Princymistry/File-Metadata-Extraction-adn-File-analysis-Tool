ENTROPY SCALE (0.0 - 8.0)
═══════════════════════════════════════════════════════════════════════════════

🟢 0.0 - 1.5  │ EMPTY/UNIFORM
             │ • Blank, solid color (1 pixel repeated)
             │ • Nearly empty files
             │ ✓ Normal and expected

🟢 1.5 - 3.5  │ UNCOMPRESSED TEXT
             │ • Plain text files (.txt, .csv, .md)
             │ • Source code (.py, .js, .cpp, .java)
             │ • Markup (.xml, .html, .json)  
             │ ✓ Expected: Natural language patterns

🟡 3.5 - 5.0  │ STRUCTURED DATA
             │ • Office docs (.docx, .xlsx, .pptx)
             │ • PDF documents
             │ • Formatted data with tags/structure
             │ ✓ Expected: Mixed text and formatting

🟡 5.0 - 6.0  │ MIXED CONTENT
             │ • Photos (.jpg, .png, .bmp typical)
             │ • Light compression archives
             │ • Mixed file types
             │ ✓ Expected: Natural variation

🟠 6.0 - 7.0  │ COMPRESSED/BINARY
             │ • Audio (.mp3, .wav, .flac, .ogg)
             │ • Video (.mp4, .avi, .mkv)
             │ • ZIP with binary files
             │ ✓ Expected: Compression algorithm

🔴 7.0 - 7.5  │ HIGH COMPRESSION
             │ • Executables (.exe, .dll, .elf)
             │ • Strong compression (RAR, 7z)
             │ • Packed/obfuscated code
             │ ⚠️  Check: Is this expected?

🔴 7.5 - 8.0  │ MAXIMUM ENTROPY
             │ • Encrypted files (.gpg, .aes)
             │ • Password-protected archives
             │ • Random data / noise
             │ • Malware packing
             │ 🚨 ALERT: Requires special analysis

ACTION GUIDE BY ENTROPY LEVEL
═══════════════════════════════════════════════════════════════════════════════

🟢 LOW ENTROPY (< 4.0)
   ✓ Archive safely
   ✓ Index for searching
   ✓ Extract text if needed
   → No security concerns

🟡 MEDIUM ENTROPY (4.0 - 6.0)
   ✓ Verify file extension matches content
   ✓ Check metadata
   ✓ Extract if archive
   → Standard files, normal operations OK

🟠 HIGH ENTROPY (6.0 - 7.5)
   ⚠️  Verify digital signature
   ⚠️  Scan with antivirus
   ⚠️  Check source/origin
   → Compressed or binary, standard analysis needed

🔴 MAXIMUM ENTROPY (7.5 - 8.0)
   🚨 FLAG AS SUSPICIOUS
   🚨 Extract for special analysis if possible
   🚨 Request password/decryption key
   🚨 Isolate from other systems
   🚨 Scan for malware signatures
   → Encrypted/Obfuscated, requires expert analysis

COMMON MISMATCHES TO WATCH FOR
═══════════════════════════════════════════════════════════════════════════════

🚩 File.txt but entropy 7.8
   → Encrypted text or fake extension
   → ACTION: Request password

🚩 Document.pdf but entropy 3.5  
   → Scanned PDF (uncompressed images)
   → ACTION: Extract images if needed

🚩 Photo.jpg but entropy 7.9
   → Encrypted image or wrong format
   → ACTION: Check magic bytes

🚩 Archive.zip but entropy 2.8
   → Probably contains text documents
   → ACTION: Extract and scan contents

🚩 Program.exe but entropy 3.5
   → Script disguised as executable
   → ACTION: Check magic bytes and inspect

KEY FACTS ABOUT ENTROPY
═══════════════════════════════════════════════════════════════════════════════

✓ Plain text has LOW entropy (patterns repeat)
✓ Compressed data has HIGH entropy (randomized bytes)
✓ Encrypted data has MAXIMUM entropy (appears random)
✓ Binary code has MEDIUM to HIGH entropy
✓ Natural images have MEDIUM entropy (color variation)
✓ Entropy says NOTHING about maliciousness
✓ High entropy ≠ Bad (legitimate compression/encryption)
✓ Low entropy ≠ Safe (could be data hiding in plain sight)
